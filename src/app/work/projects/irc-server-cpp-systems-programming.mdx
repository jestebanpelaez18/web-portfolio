## Overview

A full IRC server built from scratch in C++, following the IRC 
protocol specification and tested against IRSSI as the reference 
client. The project covers TCP/IP networking, concurrent client 
management, and protocol-level command handling, all without forking 
and using a single poll() call for all I/O operations.

## Key Features

- Managed multiple simultaneous client connections using non-blocking 
  I/O with a single poll() call, handling read, write, and listen 
  operations without blocking.
- Implemented full IRC authentication flow including PASS, NICK, and 
  USER commands with proper connection lifecycle management.
- Built channel management supporting JOIN, PART, PRIVMSG, TOPIC, 
  KICK, INVITE, and MODE commands with operator and regular user roles.
- Implemented channel modes including invite-only, topic restrictions, 
  channel keys, operator privileges, and user limits.
- All communication done over TCP/IP with IRSSI as the reference 
  client for protocol compliance testing.

## Technologies Used

- C++ (C++11)
- TCP/IP sockets
- Non-blocking I/O with poll()
- Linux
- IRC protocol (RFC 1459, RFC 7194)

## Challenges and Learnings

The most demanding aspect was managing concurrent state across 
multiple clients and channels without forking, using only a single 
poll() instance. Implementing the full IRC protocol correctly required 
careful reading of the RFC specs and systematic debugging of edge 
cases in message parsing and connection handling.

## Outcome

A fully functional IRC server that supports multiple concurrent 
clients, passes protocol compliance testing with IRSSI, and 
implements the complete set of channel operator commands.
