## Overview

This project delivers a production-style data platform using Helsinki public transport data (HSL GTFS). I implemented ingestion, transformation, analytics modeling, and machine learning as a complete pipeline that runs in containerized services.

## Key Features

- Built a Bronze, Silver, Gold architecture with raw data landing in Azure Blob Storage.
- Orchestrated end-to-end ETL with Airflow DAGs across Docker Compose services.
- Implemented PySpark jobs to parse nested Protobuf feeds and load cleaned records into PostgreSQL with idempotent writes.
- Modeled Gold analytical tables with dbt and added automated data quality checks across more than 100k records.
- Trained a SparkML Random Forest classifier to predict skipped bus stops and achieved AUC 0.92.

## Technologies Used

- Python
- PySpark
- PostgreSQL
- Airflow
- dbt
- Azure Blob Storage
- Docker Compose
- SparkML

## Challenges and Learnings

Main challenges included handling schema drift in nested transport feeds and guaranteeing pipeline idempotency between reruns. Building reliable orchestration and quality checks improved my approach to production data operations and monitoring-first engineering.

## Outcome

The result is a repeatable and observable data platform that supports both analytics and ML workflows, from raw ingestion to business-level insights and predictive modeling.
