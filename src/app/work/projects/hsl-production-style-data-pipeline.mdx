## Overview

A production-style data platform built on Helsinki public transport 
data from the HSL GTFS Realtime API. The project covers the full 
data lifecycle, from raw ingestion to analytical modeling and machine 
learning, running across containerized services with no manual 
intervention.

## Key Features

- Built a Bronze to Silver to Gold architecture with raw data landing 
  in Azure Blob Storage following Azure Data Lake patterns.
- Orchestrated end-to-end ETL with Airflow DAGs across Docker Compose 
  services, with backfill support and retry strategies for reliable 
  reprocessing.
- Implemented PySpark jobs to parse nested Protobuf feeds and load 
  cleaned records into PostgreSQL with idempotent writes.
- Modeled Gold analytical tables with dbt and implemented automated 
  data quality checks across more than 100k records, covering null 
  checks, range validation, and consistency rules.
- Trained a SparkML Random Forest classifier to predict skipped bus 
  stops, achieving AUC 0.92 on historical trip data.

## Technologies Used

- Python
- PySpark
- PostgreSQL
- Apache Airflow
- dbt
- Azure Blob Storage
- Docker Compose
- SparkML

## Challenges and Learnings

Main challenges included handling schema drift in nested transport 
feeds and guaranteeing pipeline idempotency between reruns. Building 
reliable orchestration, backfill support, and quality checks improved 
my approach to production data operations and observability-first 
engineering.

## Outcome

A repeatable and observable data platform that supports both analytics 
and ML workflows, from raw ingestion to business-level insights and 
predictive modeling.
